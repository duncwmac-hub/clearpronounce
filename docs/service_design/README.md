# ClearPronounce – Early service design audit

**Version:** 1.0  
**Scope:** Pre–/early build alignment. Value proposition, actors, journeys, touchpoints, and opportunities.  
**Status:** Living document; update as the service evolves.

---

## Purpose

This audit aligns **what ClearPronounce is as a service**, **for whom**, and **how value is created** before further feature work. It is intended to:

- Give the team a single shared map of the service.
- Make explicit the primary and secondary users and their jobs to be done.
- Identify touchpoints and front-stage/back-stage boundaries.
- Surface opportunities and risks so roadmap choices are intentional.

It is **not** a post-launch, evidence-based audit (that comes after real usage). It is a **strategic alignment** artefact.
