# Language style for Polish users

When generating or writing text for Polish users, especially at the beginning of the user journey, use **extremely simple, direct language**.

---

## Principles

1. **Prefer everyday Polish** over academic or technical vocabulary.
2. **Use short, clear sentences.**
3. **Avoid linguistic terminology** such as *fonem*, *transkrypcja*, *IPA*, etc., unless the product has already introduced and explained them.
4. **Write so an average Polish speaker understands immediately**, without explanation.
5. **For pronunciation hints**, use natural instructions:
   - **"Powiedz:"** or **"Wymawiaj:"**  
   Do **not** use abbreviations like *czyt.* or technical notation without explanation.

---

## Example

**Good:**

> ClearPronounce (Powiedz: *KLIR-pronałns*)

**Avoid:**

> ClearPronounce (IPA: /klɪər prənaʊns/), czyt. transkrypcja fonetyczna…

---

## Goal

The user should feel that the system is **simple and obvious**, not academic.

**Tone:** Conversational, confident, and practical.

---

## When to use this

- In-app copy and instructions for Polish-speaking learners.
- Onboarding, tooltips, and help text.
- Any generated or static text that addresses the learner in Polish.
- Pronunciation hints and respelling labels shown to users.

Update this doc when we add new Polish strings or refine the style.
